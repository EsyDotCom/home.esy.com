# Workflow URL And Naming Standard

## Standard

Use `/workflows` as the canonical public route for Esy's workflow library.

Do not use `/templates` as the primary route. "Templates" alone sounds like static documents, design assets, or prompt packs. Esy is positioning users around workflows: repeatable processes that agents can run.

## User-Facing Labels

Use these labels consistently:

- Page label and breadcrumb: **Agentic Workflows**
- Section heading: **Workflow Templates**
- Card badge: **Workflow Template**
- Body copy: **workflow template**

Avoid standalone **Templates** for workflow pages, cards, breadcrumbs, and CTAs.

## Prompt Positioning

Workflow pages can mention AI prompts when explaining the behavior shift:

> Move beyond one-off AI prompts. Choose a workflow template and let Esy guide the research, writing, review, and formatting.

Keep prompt-focused language only for legacy prompt pages or pages that are explicitly about prompts.

## SEO Rationale

`/workflows` owns Esy's product category. "Workflow template" keeps the SEO value of "template" while making clear that the object is a runnable process, not a static prompt.
